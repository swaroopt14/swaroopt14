# Code of Conduct

## Purpose

This project is intended to be a respectful, professional, and collaborative space for everyone involved in building, reviewing, testing, documenting, and discussing Arealis Zord.

We want contributors, maintainers, users, and reviewers to feel safe participating without harassment, hostility, or exclusion.

## Expected Behavior

When participating in this project, please:

- Be respectful in technical and non-technical discussions
- Assume good intent, while still giving honest and direct feedback
- Focus on improving the work rather than attacking people
- Welcome questions from contributors with different experience levels
- Be thoughtful when discussing security, compliance, customer data, and operational risks
- Respect differences in background, identity, communication style, and expertise
- Accept correction gracefully and help keep discussions productive

## Unacceptable Behavior

The following behavior is not acceptable in project spaces, whether public or private:

- Harassment, intimidation, threats, or personal attacks
- Discriminatory, hateful, or demeaning language
- Trolling, deliberate provocation, or repeated bad-faith argument
- Sharing private, confidential, or sensitive information without permission
- Sexualized language, imagery, or unwelcome advances
- Doxxing or attempts to expose another person's private information
- Repeated disruption of discussions, reviews, planning, or support channels
- Retaliation against someone who reports a concern or helps enforce this Code of Conduct

## Scope

This Code of Conduct applies to project-related spaces, including:

- GitHub issues, pull requests, commits, discussions, and reviews
- Documentation and design discussions
- Chat, email, or coordination channels used for this project
- Operational, deployment, and incident-related collaboration tied to this repository

## Reporting

If you experience or witness behavior that violates this Code of Conduct, report it through your existing private maintainer channel for this repository, or contact the repository owners directly.

When reporting, include:

- What happened
- Where it happened
- When it happened
- The people involved, if known
- Any screenshots, logs, or supporting context you can safely share

Please avoid posting sensitive conduct reports publicly when a private report is possible.

## Maintainer Responsibilities

Project maintainers may:

- Clarify expectations for behavior
- Ask for language or conduct to change
- Remove or moderate harmful content
- Pause, lock, or limit discussions that are no longer productive
- Temporarily or permanently restrict participation when necessary to protect the project community

Maintainers will aim to review reports fairly, keep shared information as limited as practical, and act in proportion to the severity and pattern of behavior.

## Enforcement Guidelines

Possible responses may include:

- A private warning
- A request to stop specific behavior
- Removal of comments, posts, or other content
- Temporary suspension from project spaces
- Permanent removal from participation in this repository or related collaboration spaces

## Security And Safety

If a report includes threats, harassment, stalking, credible safety concerns, or exposure of secrets or customer data, maintainers may prioritize immediate protective action before a full review is completed.

Security vulnerabilities should still be reported through the process described in [SECURITY.md](./SECURITY.md).

## Changes

The maintainers may update this Code of Conduct as the project evolves. Significant updates should be committed in the repository so contributors can review them openly.
